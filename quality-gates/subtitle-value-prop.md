# Quality Gate: Subtitle Value Proposition

## Rule

The subtitle MUST communicate a user benefit or value proposition, not just a list of keywords.

```
GOOD: "Home Monitor & Baby Cam"       <- benefit: monitoring + baby safety
GOOD: "Learn 40+ Languages Free"      <- benefit: breadth + price
GOOD: "Scan, Sign & Send Documents"   <- benefit: workflow actions

BAD:  "Camera Security Monitor Home"  <- keyword dump
BAD:  "Photo Video Editor Filter"     <- keyword dump
BAD:  "Best Free VPN Proxy Fast"      <- keyword dump with superlatives
```

## Why

1. **Conversion**: The subtitle appears directly below the title in search results. It's your 30-character elevator pitch. A value-proposition subtitle answers "Why should I download this?" — a keyword dump answers nothing.
2. **Double duty**: A well-crafted subtitle can include 2-3 keywords while ALSO communicating value. "Home Monitor & Baby Cam" contains "home", "monitor", "baby", and "cam" — all rankable — while telling the user exactly what the app does.
3. **Differentiation**: In a search result with 10 apps, the one with a clear subtitle stands out from the keyword-stuffed competitors.

## How to check

1. Read the subtitle as a phrase — does it communicate what the app does or why it's valuable?
2. Check for action verbs, benefit words, or clear feature descriptions
3. Red flags: 4+ disconnected keywords, no verbs/prepositions, reads like a tag cloud
4. **FAIL** if the subtitle is 4+ consecutive keywords with no connecting words
5. **WARN** if the subtitle doesn't clearly communicate a benefit

## Exceptions

- Extremely well-known app categories where the feature list IS the value prop (rare)

## Fix

1. Identify the top 2-3 keywords you want in the subtitle
2. Craft a natural phrase that includes those keywords: `[Action/Feature] [Connector] [Feature]`
3. Use connecting words sparingly: "&", "+", "and", "-" are fine (they're stripped during indexing anyway)
4. Test: show the title + subtitle to someone unfamiliar with the app. Can they tell what it does in 2 seconds?
